# DEON_SITE_ARCHITECTURE.md

This document is the authoritative source of truth for the DEON website information architecture.

Navigation, breadcrumbs, side panels, market context bars, mega menus, routing structures, and content relationships must be generated from this architecture.

Do not simplify.

Do not flatten.

Do not remove hierarchy levels.

Do not infer alternative structures.

If a navigation item exists in this document, it should exist in the website.

---

# Navigation Philosophy

The navigation is not merely a collection of links.

It is a visual representation of the DEON ecosystem.

A user should be able to understand:

- What markets exist
- What segments exist
- What applications exist
- What product families exist
- What resources exist

without visiting every page individually.

The navigation should communicate information architecture.

# BREADCRUMB ARCHITECTURE

The breadcrumb is part of the information architecture.

It should appear directly below the global navigation.

Examples:

Home > Markets > Electronics

Home > Markets > Electronics > PCB Assembly

Home > Markets > Electrical & Insulation > Transformers

Home > Products > PVC Electrical Tapes

Home > Resources > Technical Guides

---

# BREADCRUMB BEHAVIOR

Every parent level should be clickable.

Examples:

Home > Markets > Electronics > PCB Assembly

Behavior:

- Home → Homepage
- Markets → Markets landing page
- Electronics → Electronics page
- PCB Assembly → Current page

The current page should be visually distinguished.

Breadcrumbs are navigation components.

Not decorative text.

---

# UX CONSISTENCY RULE

Electronics is the benchmark implementation.

All market pages, segment pages, application pages, product pages, resource pages, and capability pages should inherit the same:

- Header
- Navigation
- Market Context Bar
- Breadcrumb Pattern
- Mega Menu Behavior
- Hero Structure
- Section Rhythm
- Visual Hierarchy
- CTA Placement
- Footer

There must only be one design language across the entire ecosystem.

The user should never feel that different sections of the site were built by different systems.
